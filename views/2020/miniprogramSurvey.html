      # 历史沿革

      # 移动三剑客

      HTML5 于 2007 年在 W3C 立项，与 iPhone 发布同年。

      乔布斯曾期待 HTML5 能帮助 iPhone 打造起应用生态系统。但 HTML5 的发展速度并不如预期，于是在 iPhone 站稳脚跟后，发布了自己的 App Store，开启了移动互联网的原生应用时代。

      随后的 Android，本来是基于 Linux 的 OS，采用 C + HTML5 的双模应用生态策略，然而 C 的开发难度太大，HTML5 体验又不行。后作出调整，Android 依靠 Java 技术生态，在竞争中脱颖而出。

      于是乎，在移动互联网初期，应用生态被定了基调 —— 原生开发，即 IOS 开发 / Android 开发（Native App）

      但是 HTML 有它最为显著的优点：免安装、即点即用，这是用户非常青睐的

      # 国内萌芽

      国内有一批做浏览器的厂商，尝试去改进 HTML5，通过给 WebView 扩展原生能力，补充 JS API，让 HTML5 应用可以实现更多功能。

      这类业务发展的顶峰，是微信的 JS SDK。微信为它的浏览器内核扩充了大量 JS API，让开发者可以用 JS 调用微信支付、扫码等众多 HTML5 做不到的功能。

      但是每次点击要等半天白屏（H5 需要加载DOM、样式、逻辑代码耗时），业内开始寻找新的解决方案。。。

      # 技术本质

      随着技术的演变，Hybrid App（介于原生app 和web app） 取得了性能上的突破。

      业内有人（DCloud）提出：把之前 Hybrid 应用里的运行于客户端的 JS 代码，先打包发布到服务器，制定流式加载协议，手机端引擎动态下载这些 JS 代码到本地，并且为了第一次加载速度更快，实现了应用的边下载边运行。

      就像流媒体的边下边播一样，应用也可以实现边用边下。

      小程序让 JS 应用功能体验达到原生。

      ps:

      移动应用开发演变成目前三种方式：Native App、Hybrid App、Web App

      类型 释义
      Native App IOS / Android 原生开发
      Hybrid App C/S 的应用在每次页面加载时，仅需要联网获取 JSON 数据；
      Web App B/S 应用除了 JSON 数据外，还需要每次从服务器加载页面 DOM、样式、逻辑代码，所以 B/S 应用的页面加载很慢，体验很差。

      # 引爆生态

      中国独特的开发生态：小程序。

      2017 年 1 月 9 日，微信公开课，小程序面向用户正式推出。

      2017 年 9 月 20 日，支付宝小程序正式上线。

      2018 年 7 月 4 日，百度小程序正式上线。

      2018 年 11 月 17，今日头条小程序正式上线。

      阿拉丁发布的2019小程序互联网发展白皮书表明，全网小程序数量已超过 450 万个，小程序的日活用户数到达 3.3 亿，是无可争议的一级流量入口。

      # 框架QA

      Q: 为什么使用第三方框架?

      A: 上手快、支持多端编译、组件化开发、支持 npm、支持更多 ES6 特性、生命周期补充、支持样式编辑器（Scss/Less）、支持代码编译器（Babel/Typescript）

      现在主流的小程序开发框架：Taro、WePY 、uni-app、mpvue、chameleon、Megalo

      ps:基于 Vue 技术栈的有uni-app、mpvue、chameleon、Megalo

      1. 开发规范-重要
      • 页面文件遵循 Vue 单文件组件 (SFC) 规范
      • 组件标签靠近小程序规范，详见uni-app 组件规范
      • 接口能力（JS API）靠近微信小程序规范，但需将前缀 wx 替换为 uni，详见uni-app接口规范
      • 数据绑定及事件处理同 Vue.js 规范，同时补充了App及页面的生命周期
      • 为兼容多端运行，建议使用flex布局进行开发

      备注：

      生命周期：应用生命周期（例：onLaunch）、页面生命周期（例：onShow）、组件生命周期（同 vue），官方文档有详述

      1. 重点突破
      • pages.json（配置路由、全局样式、tabBar、分包等）。
      • 使用 vue 注意事项，看官网及常见问题，有详细文档。
      • 封装的 API 请查看官网文档，示例：uni.request()、uni.navigateTo()、uni.makePhoneCall()等。
      • uniapp 有自己的一套 UI 组件。
      • 插件市场带来更多选择。
