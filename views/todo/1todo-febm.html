      # 缘起

      做前端开发至今，收藏过不少网站链接作为浏览器书签。随着时间的推移，收集的链接越来越多，书签的整理难度越来越大，冗余和不规范也越来越多。

      回首一看，已收集了上千书签链接。

      让人不禁思考，能不能做点什么,来规范在 前端路径 (opens new window) 学习过程中收集的书签？

      1. 可以让自己在路径学习中更加系统化。
      2. 遇到问题（总有一些共通且常见的问题）可以快速定位到一个解决链接（也能快速地分享给同伴）。
      3. 一串简单的 URL，就能将你传送到一个新的世界，打开视野。

      之前在掘金上发布过一篇文章 作为一个卑微的前端仔，我收集了哪些网站链接？ (opens new window)。每隔一段时间，都会新增一两个赞，让我想兑现结语的那段话：

      后续仍会持续更新，您的点赞是我坚持下去的动力。或将更改展现形式，以期梳理自我和方便大家。期待交流和建议！

      所以后来自己又尝试了用网站的形式展示这些链接，但是效果并不满意（设计、维护等无力）。如有兴趣，可以本地下载了跑一跑：项目地址-FEMB (opens new window)，有更多想法欢迎交流~ +V: ifuckinghigh

      一个人的力量实在太小，且水平（号召力、组织力）有限，只得抛砖引玉。

      最终希望实现：前端工程师们可以将各自浏览器书签添加到这个项目中，共同努力，做一个规范、有效的“前端书签”网站。

      一切就止步于此吗？事情不能拖，拖着拖着想法都要给拖没了。不完美前进吧，故撰此文以续想法，邀志同者共进。

      很喜欢李宗盛《山丘》里的一句歌词：“说不定我一生涓滴意念，侥幸汇成河”。

我们一生中有很多散乱的意念，有的人能侥幸将其汇聚成河，有的人则随风将其遗忘在了时间里。

意义或许就是没有意义，只是觉得自己想这么去做！

      # 撰文说明

      1. 每个网站都是本瓜或多或少亲自体验过的，承诺不存在无脑平行搬运的情况。
      2. 👍点赞是本瓜结合体验的主观判断，仅作参考。从三个赞到六个赞可依次理解为：不错、很好、太棒了、我佛了。
      3. 我不否认对于链接分类会存在认知偏差，如有意见可反馈讨论。

      # 格式说明

      1. 平台类型的书签格式是：链接 + 平台名 + 备注 + 评赞。
      2. 文章类型的书签格式是：文章名 + 备注 + 评赞。
      3. 平台名、备注为非必填项。
